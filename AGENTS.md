# AGENTS.md

## Website Development

When working on the HOTFIX website (HTML, CSS, React components, pages, layouts), you **must** follow the design system defined in `DESIGN.md`.

- Read `DESIGN.md` before making any visual or UI changes.
- Use the color tokens, typography scale, spacing system, and component specs from `DESIGN.md` — not ad-hoc values.
- Respect the dark-canvas aesthetic, surface ladder, and red-orange accent rules.
- If `DESIGN.md` specifies a component style (button, card, nav, etc.), match it exactly.
- If no component spec exists for what you need, follow the closest existing pattern and the general principles in `DESIGN.md`.

## General

- Keep code clean, well-structured, and consistent with existing patterns in the codebase.
- Prefer editing existing files over creating new ones.
- Use TypeScript throughout.
